## Pattern 1: Create the App

**INCORRECT:** Using `--tailwind` flag — it installs Tailwind v3 with a config file. This project uses v4 (CSS-only, no config file).

**CORRECT — scaffold without Tailwind, add deps manually:**

```bash
npx create-next-app@latest site \
  --typescript \
  --app \
  --src-dir=false \
  --tailwind=false \
  --eslint \
  --import-alias "@/*"
cd site

npm install \
  @commercetools/platform-sdk \
  @commercetools/sdk-client-v2 \
  next-intl \
  swr \
  jose \
  tailwindcss @tailwindcss/postcss postcss
```

**Dependency rationale:**

| Package | Purpose |
|---------|---------|
| `@commercetools/platform-sdk` | Typed CT REST API client |
| `@commercetools/sdk-client-v2` | Token management + HTTP middleware for the SDK |
| `next-intl` | Locale routing and message translation |
| `swr` | Client-side data fetching with cache/revalidation |
| `jose` | JWT signing/verification for sessions (server-only) |
| `tailwindcss @tailwindcss/postcss postcss` | Tailwind v4 — CSS-first, no config file |

## Pattern 3: next-intl Locale Routing

**INCORRECT:** Resolving locale from cookies inside `getRequestConfig`. next-intl v4 uses `requestLocale` (from the URL segment) as the primary source — cookies are a fallback for the middleware redirect, not for the request config.

**CORRECT — two-file setup:**

`i18n/routing.ts`:
```typescript
import { defineRouting } from 'next-intl/routing';
import { createNavigation } from 'next-intl/navigation';

export const routing = defineRouting({
  locales: ['en-us', 'en-gb', 'de-de'],
  defaultLocale: 'en-us',
  localePrefix: 'always', // every URL has /en-us/, /de-de/, etc.
});

// Use these instead of next/link, next/navigation in locale-aware code
export const { Link, redirect, usePathname, useRouter, getPathname } =
  createNavigation(routing);
```

`i18n/request.ts`:
```typescript
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !routing.locales.includes(locale as (typeof routing.locales)[number])) {
    locale = routing.defaultLocale;
  }
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

`next.config.ts`:
```typescript
import type { NextConfig } from 'next';
import createNextIntlPlugin from 'next-intl/plugin';

const withNextIntl = createNextIntlPlugin('./i18n/request.ts');

const nextConfig: NextConfig = {
  images: {
    unoptimized: true, // external CDN handles optimisation
    remotePatterns: [
      { protocol: 'https', hostname: 'storage.googleapis.com' },
      { protocol: 'https', hostname: '**' },
    ],
  },
};

export default withNextIntl(nextConfig);
```

> **Locale format:** URL segments use lowercase-hyphen: `en-us`, `de-de`. CT API calls use `en-US`, `de-DE`. Keep a mapping in `lib/utils.ts`.

## Pattern 4: Locale Middleware

**INCORRECT:** No middleware — users hitting `/` get a 404 because the app only handles `/en-us/`, `/de-de/`, etc.

**CORRECT — `proxy.ts` at project root redirects bare paths to locale-prefixed ones:**

```typescript
// proxy.ts  (Next.js treats this as middleware via the config.matcher export)
import { NextRequest, NextResponse } from 'next/server';
import { DEFAULT_LOCALE } from '@/lib/utils';

const COUNTRY_TO_LOCALE: Record<string, string> = {
  US: 'en-us',
  GB: 'en-gb',
  DE: 'de-de',
};
const LOCALES = Object.values(COUNTRY_TO_LOCALE);
const DEFAULT_URL_LOCALE = COUNTRY_TO_LOCALE[DEFAULT_LOCALE.country];

export function proxy(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // Skip API routes, Next.js internals, and static files
  if (
    pathname.startsWith('/api/') ||
    pathname.startsWith('/_next/') ||
    pathname.startsWith('/favicon') ||
    pathname.includes('.')
  ) {
    return NextResponse.next();
  }

  // Already locale-prefixed — set the x-next-intl-locale header and pass through
  const matchedLocale = LOCALES.find(
    (l) => pathname === `/${l}` || pathname.startsWith(`/${l}/`)
  );
  if (matchedLocale) {
    const response = NextResponse.next();
    response.headers.set('x-next-intl-locale', matchedLocale);
    return response;
  }

  // No locale prefix — redirect based on vibe-country cookie
  const country = request.cookies.get('vibe-country')?.value || 'US';
  const locale = COUNTRY_TO_LOCALE[country] || DEFAULT_URL_LOCALE;
  const url = request.nextUrl.clone();
  url.pathname = `/${locale}${pathname === '/' ? '' : pathname}`;
  return NextResponse.redirect(url);
}

export const config = {
  matcher: ['/((?!api|_next|favicon|.*\\..*).*)', '/'],
};
```

> **Why `proxy.ts` and not `middleware.ts`?** The filename is arbitrary — Next.js identifies middleware by the `config.matcher` export. `proxy.ts` is the convention used in this project.

## Pattern 5: Folder Structure

**INCORRECT:** Putting CT logic in components, or hooks in `app/`. This breaks the BFF boundary and creates implicit dependencies.

**CORRECT — strict separation by responsibility:**

```
site/
├── app/
│   ├── [locale]/           ← locale-aware pages (Server Components by default)
│   │   ├── layout.tsx      ← wraps children in NextIntlClientProvider + CartProvider
│   │   └── page.tsx
│   ├── api/                ← BFF Route Handlers (server-only, never imported client-side)
│   ├── globals.css
│   └── layout.tsx          ← root layout (passes children through)
├── lib/
│   ├── ct/                 ← CT SDK calls (server-only — never import in 'use client' files)
│   │   ├── client.ts       ← apiRoot singleton
│   │   ├── cart.ts
│   │   ├── categories.ts
│   │   ├── auth.ts
│   │   └── search.ts
│   ├── mappers/            ← CT SDK types → internal domain types
│   │   ├── product.ts
│   │   └── category.ts
│   ├── types.ts            ← shared domain types (Product, Category, Variant, Price)
│   ├── cache-keys.ts       ← SWR cache key constants
│   ├── session.ts          ← JWT helpers: getSession, createSessionToken, setSessionCookie
│   └── utils.ts            ← formatMoney, getLocalizedString, COUNTRY_CONFIG
├── hooks/                  ← client-side SWR hooks ('use client')
├── components/
│   ├── ui/                 ← Button, Input, Drawer (design system)
│   ├── layout/             ← Header, Footer, MiniCart
│   └── product/            ← ProductCard, ProductGrid, AddToCartButton
├── context/
│   └── CartContext.tsx     ← CartProvider + useCartContext
├── i18n/
│   ├── routing.ts
│   └── request.ts
├── messages/
│   ├── en-us.json
│   ├── en-gb.json
│   └── de-de.json
└── proxy.ts                ← middleware
```

## Pattern 6: Shared Types and Utilities

**INCORRECT:** Importing CT SDK types directly in components: `import type { ProductProjection } from '@commercetools/platform-sdk'`.

**CORRECT — all frontend code imports from `lib/types.ts`:**

`lib/types.ts` (excerpt):
```typescript
export interface Price {
  centAmount: number;
  currencyCode: string;
  discounted?: { centAmount: number; currencyCode: string };
}

export interface Variant {
  id: number;
  sku: string;
  images: string[];
  price?: Price;
  prices: Price[];
  attributes: Array<{ name: string; value: unknown }>;
  availability?: { isOnStock?: boolean };
}

export interface Product {
  type: 'Product';
  id: string;
  name: string;
  slug: string;
  description?: string;
  categories: Array<{ id: string }>;
  variants: Variant[]; // index 0 is the master variant
}
```

`lib/utils.ts` (key exports):
```typescript
export const COUNTRY_CONFIG: Record<string, { currency: string; locale: string }> = {
  US: { currency: 'USD', locale: 'en-US' },
  GB: { currency: 'GBP', locale: 'en-GB' },
  DE: { currency: 'EUR', locale: 'de-DE' },
};

export const DEFAULT_LOCALE = { country: 'US', currency: 'USD', locale: 'en-US' };

export function formatMoney(centAmount: number, currencyCode: string, locale = 'en-US'): string {
  return new Intl.NumberFormat(locale, { style: 'currency', currency: currencyCode })
    .format(centAmount / 100);
}

export function getLocalizedString(obj: Record<string, string> | undefined, locale: string): string {
  if (!obj) return '';
  return obj[locale] ?? obj[locale.split('-')[0]] ?? Object.values(obj)[0] ?? '';
}
```

`lib/cache-keys.ts`:
```typescript
export const KEY_CART = 'cart';
export const KEY_ACCOUNT = 'account';
export const KEY_ORDERS = 'orders';
export const KEY_ADDRESSES = 'addresses';
export const KEY_WISHLIST = 'wishlist';
export function keyOrder(id: string) { return `order-${id}`; }
```
